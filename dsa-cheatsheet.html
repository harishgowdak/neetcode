The DSA Cheatsheet

When · to · Use · What

A practical field guide to picking the right tool — organized by the shape of the problem, not the shape of the answer.

Choosing a data structure is really choosing what you want to be fast. Every structure trades something away — arrays give up cheap insertion to win on random access; hash maps give up ordering to win on lookup; trees give up constant-time operations to keep things sorted. The trick is recognizing which tradeoff the problem actually rewards. Most of the time the clue is in the verbs: find, count, order, connect, shortest, all possible. Read the verbs, pick the structure, then worry about the algorithm.

01

Data Structures

pick by access pattern
Array / List
Default choice

Fast index access, predictable size. Start here unless you have a reason not to.

access O(1) · insert-mid O(n)
Hash Map
"Have I seen this?"

Counting, deduplication, memoization, lookup-by-key. No ordering guarantees.

get/set O(1) avg · worst O(n)
Tree Map
Ordered lookup

When you need sorted keys, range queries, or "smallest key ≥ X".

all ops O(log n)
Stack
LIFO · last-in-first-out

Undo, bracket matching, DFS, expression parsing, "next greater element".

push/pop O(1)
Queue / Deque
FIFO · first-in-first-out

BFS, scheduling, sliding windows (deque), producer-consumer pipelines.

enq/deq O(1)
Heap
Top-K · priority

Always process the highest-priority item next. Dijkstra, median, merge-k-sorted.

push/pop O(log n) · peek O(1)
Trie
Prefix problems

Autocomplete, spell-check, IP routing, word games. Overkill without prefix needs.

lookup O(k) where k = key length
Union-Find
Grouping · connectivity

"Same component?" Kruskal's MST, cycle detection in undirected graphs.

union/find ~O(α(n)) ≈ O(1)
Graph
Relationships · networks

Adjacency list for sparse (most real cases), matrix for dense or O(1) edge checks.

list: space O(V+E) · matrix O(V²)
Linked List
Rarely the answer

Only when O(1) insert/delete at known nodes matters and random access doesn't. Poor cache behaviour.

insert-at-node O(1) · access O(n)
Segment / Fenwick Tree
Range query + update

Sum/min/max over [l, r] while values change. Competitive programming staple.

query/update O(log n)
Hash Set
Membership only

"Is X in this collection?" Cheaper than a map when you don't need values.

contains O(1) avg
02

Algorithms

pick by problem shape
Two Pointers
Sorted arrays · pairs

Find pairs/triplets with a target, remove duplicates in place, palindrome checks.

typically O(n)
Sliding Window
Contiguous sub-range

"Longest substring with…", "max sum of window k", "min window containing…".

O(n) with amortized moves
Binary Search
Sorted · monotonic

Sorted arrays, or "search the answer space" — smallest value satisfying a condition.

O(log n)
DFS
Go deep · explore

Tree traversal, backtracking, topological sort, cycle detection, connected components.

O(V + E)
BFS
Shortest · unweighted

Level-order, "minimum steps to reach", shortest path in unweighted graphs.

O(V + E)
Dijkstra
Shortest · weighted

Shortest path with non-negative edge weights. Uses a min-heap.

O((V+E) log V)
Dynamic Programming
Overlapping subproblems

"Count the ways", "min/max cost to", "can you reach". Define state, then transition.

varies · often O(n²) or O(n·k)
Greedy
Local → global

When a local optimal choice provably wins. Prove it works — don't hope. Interval scheduling, Huffman.

often O(n log n) after sorting
Backtracking
All possibilities

Generate permutations, combinations, N-queens, Sudoku. Small input sizes only.

exponential · prune aggressively
Divide & Conquer
Split · solve · merge

Merge sort, quick sort, closest pair of points. Independent subproblems.

often O(n log n)
Bellman–Ford
Negative weights

Shortest path when edges can be negative. Detects negative cycles.

O(V · E)
Floyd–Warshall
All-pairs shortest

All-pairs shortest paths on small dense graphs. Three nested loops, famously concise.

O(V³)

Problem Signals

if you see X, reach for Y
"Top K" / "K largest"
→ Heap of size K
"Seen before?" / "Count of…"
→ Hash map or set
"Contiguous subarray/substring"
→ Sliding window
"Sorted array, find pair…"
→ Two pointers
"Shortest path" · unweighted
→ BFS
"Shortest path" · weighted, non-neg
→ Dijkstra
"All possible…" · small input
→ Backtracking
"Min / max / count of ways"
→ Dynamic programming
"Connected components" / "same group?"
→ Union-Find or DFS
"Next greater / smaller element"
→ Monotonic stack
"Prefix / autocomplete"
→ Trie
"Smallest value that satisfies…"
→ Binary search on answer
"LIFO" / undo / matched pairs
→ Stack
"Scheduled / streaming / level-by-level"
→ Queue
"Range sum/min/max" + updates
→ Segment / Fenwick tree
